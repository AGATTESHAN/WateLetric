# WateLetric - Web Functionality & System Notes

This document provides a detailed overview of the structure, features, logic, and functionality of the **WateLetric Utility Management** web application.

---

## 1. Project Overview
WateLetric is an AI-powered smart utility management platform that monitors, forecasts, and manages Electricity and Water consumption for non-domestic/industrial facilities. It uses:
1. **Machine Learning (XGBoost)** to forecast future usage and calculate projected monthly bills.
2. **Dynamic ML Anomaly Detection** to flag unexpected consumption surges or water leaks in real-time.
3. **Generative AI (Google Gemini)** as a conversational assistant ("Berry Assistant") to explain bills, anomalies, and recommend optimizations.

---

## 2. File & Component Structure
* **`server.py`**: A multi-threaded Python web server.
  - Serves static frontend assets (HTML, CSS, JS, JSON) on `http://localhost:8000/`.
  - Serves as a proxy `/api/chat` for the chatbot, injecting dashboard usage contexts before calling the **Google Gemini API** (using the model `gemini-2.5-flash`).
* **`run_agent.py`**: The backend machine learning and preprocessing pipeline.
  - Trains XGBoost regressors on 30 days of historical mock data.
  - Performs predictions and anomaly detection on the live runtime data.
  - Exports calculations into `billing_data.js`.
* **`billing_data.js`**: Generated by the ML pipeline. Contains the daily stats, interval metrics, forecasts, and detected anomalies for the dashboard to read.
* **`electricity.html`**: Electricity usage dashboard.
* **`water.html`**: Water usage dashboard (includes pumping cost metrics).
* **`summary.html`**: Main utility summary dashboard comparing electricity and water.
* **`profile.html`**: Settings/user profile screen.
* **`index.html`**: Landing/Login page.

## 3. Core Functionality & Logic

### A. Machine Learning Forecasting
- **Training Dataset:** Trained on `mock_dataset_electricity_30days.json` and `mock_dataset_water_30days.json` (48 intervals of 30-minute readings per day).
- **Features Used:** `time` (hour float), `day_of_week` (mapped 0-6), `occupancy` (current occupants count), and **lag features** (`lag_1`, `lag_2`, `lag_24` (24h ago), `lag_48` (48h ago), `lag_weekly` (7 days ago)).
- **Prediction:** The model learns baseline usage under normal occupancy. When fed the live runtime data, it predicts what the expected consumption *should* be for every 30-minute interval.

### B. Dynamic Anomaly Detection
Instead of hardcoding alerts, the system dynamically calculates deviations:
$$\text{Deviation} = \text{Actual Sensor Value} - \text{XGBoost Model Prediction}$$
- **Electricity Spike:** If deviation $> 5.0\text{ kWh}$, it flags a `SPIKE DETECTED` anomaly (e.g. equipment left on).
- **Water Leak:** If deviation $> 10.0\text{ Liters/hr}$ and building occupancy is zero, it flags a `LEAK DETECTED` anomaly.
- **Alert Triggering:** Triggered anomalies immediately display a red alert badge in the UI and cause the Chatbot to dynamically post a custom warning message to the user.

### C. Tariff & Bill Calculations
The backend scales the single day of runtime data (combining actual readings up to 2:00 PM and ML predictions for the remainder of the day) to a 30-day projection, then calculates utility bills using local tariffs:
- **Electricity (TNB Tariff D - Industrial):**
  - First 200 kWh: RM 0.380 / kWh.
  - 201 kWh onwards: RM 0.441 / kWh.
  - Minimum monthly charge: RM 7.20.
- **Water (Air Selangor Non-Domestic/Commercial):**
  - First 35,000 Liters: RM 0.00351 / Liter.
  - Above 35,000 Liters: RM 0.00383 / Liter.
  - Minimum monthly charge: RM 36.00.
- **Pumping Electricity Cost (Water Dashboard):**
  - Estimates the electricity overhead required to pump water (assumes `0.05 kWh` of electrical energy per Liter of water pumped, multiplied by the average electricity rate).

## 4. User Interface Features

### A. Main Dashboards (Electricity / Water)
- **Live Consumption Ticker:** Displays the active consumption rate.
- **Consumption Cards:** Displays Projected Monthly Total, Estimated Monthly Bill, and Daily Averages.
- **Double-Layer Graph:**
  - Solid bars plot actual consumption readings (simulated up to 2:00 PM).
  - Dashed/faded bars plot predicted forecast values (after 2:00 PM) for the remainder of the day.
- **Next 30 Minutes Prediction Card:** Tells the user the exact forecast for the next interval and shows a progress bar indicating if there is a predicted usage surge compared to the historical baseline.

### B. Summary Dashboard (`summary.html`)
- Displays average utility usage for both energy and water side-by-side.
- **Peak Usage Analysis:** Dynamically evaluates the runtime intervals, identifies the exact maximum consumption peaks, and displays the formatted peak times and descriptions (e.g., Electricity peak at 2:30 PM, Water peak at 12:30 PM).

### C. Simulation Engine
On `electricity.html` and `water.html`, the user can toggle the **"Start Simulation"** button:
- Clears the graph and rewinds time to 12:00 AM.
- Sequentially plots the actual runtime sensor values onto the chart second-by-second.
- Allows the user to observe live metrics ticking up and dynamically triggers the AI bot and red badges at the exact seconds anomalies are reached.

### D. Context-Aware AI Chatbot (Berry Assistant)
- **Interactive Chat:** Users can type custom inquiries (e.g. *"Why is my water bill so high?"*).
- **Context Injection:** The frontend packages the chatbot request with the current page state (actual usage, bill estimates, forecasts, and anomaly flags).
- **API Proxy:** The backend routes the data to the Gemini API, which returns answers tailored to the exact metrics currently showing on the screen.
- **Self-Greeting:** When a dashboard loads, the assistant automatically calculates and delivers a tailored summary greeting summarizing key stats and warnings.
